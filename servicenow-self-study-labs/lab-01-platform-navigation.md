# Lab 01 – Platform Navigation

## Objective

Understand the basic structure of the ServiceNow platform and how users interact with records, lists, forms, and applications.

## Concepts

- Instance
- Application Navigator
- List View
- Form View
- Filters
- Favorites
- Records and Tables
- Service Portal

## Steps

1. Open a personal ServiceNow developer instance.
2. Use the Application Navigator to search for common modules.
3. Open an example list, such as Incidents.
4. Open one record and compare list view with form view.
5. Add frequently used modules to favorites.
6. Write down the fields visible in a typical incident form.

## Reflection

The platform is organized around tables and records. Users interact with the system mainly through lists and forms. Understanding navigation is the first step before learning ITSM processes or automation.
